## 기능 관련

### Q4: 뽀모도로 백그라운드 처리
- 앱이 백그라운드로 갈 때 타이머 유지 방법
  - A) expo-task-manager로 백그라운드 타스크 (배터리 소모)
  - B) startTime 저장 → 복귀 시 경과 시간 계산 (가볍지만 정확도 낮음)
  - C) 화면 항상 켜짐 유지 (expo-keep-awake) + 번인 방지
- **PRD 의도**: 화면 켜진 채 유지 (옵션 C에 가까움)
-> 백그라운드로 갔을때 바로 실패처리를 해야하므로 타이머 정지시켜야함. 

### Q5: 방해금지 모드 연동 범위
- A) 단순 안내만 (시스템 설정으로 이동 유도) — 가장 심플
- B) expo-intent-launcher로 DND 설정 화면 직접 열기
- C) 네이티브 모듈로 직접 DND 제어 (Managed Expo에서 제한적)
- **권장**: 옵션 B (안내 + 설정 화면 열기)
-> 권장사항대로 구현.

### Q6: OLED 번인 방지 구현

## 디자인 관련

### Q8: Skia 유지 vs React Native Image
- 현재: Skia Canvas로 모든 렌더링
- 대안: 픽셀아트는 일반 Image 컴포넌트 + Animated로도 충분할 수 있음
- **Skia 유지 장점**: 60fps 렌더링, 복잡한 이펙트 가능
- **Image 전환 장점**: 번들 크기 감소 (canvaskit.wasm 8MB 제거)
- **결정 필요**: 웹 빌드 시 WASM 의존성 유지할 것인가
-> 복잡한 이펙트 필요없으므로 이미지만 사용해보자.

### Q9: 상점 시스템 확장
- 기존: 색상 변경 (벡터)
- 신규: 스프라이트 교체 (픽셀아트)
- **결정 필요**: 스킨별 별도 스프라이트시트 vs 색상 팔레트 스왑
-> 상점은 일단 지금은 제외하고 구현. 구현계획 문서로만 남기기
